# Definition of Done

A workspace and the system as a whole are validated against this checklist.

## Workflow

- [ ] The system always asks "Hobby or Project?" before any planning work.
- [ ] The correct specialists activate for the chosen mode.
- [ ] Non-applicable specialists are marked (Not applicable / Optional / Waiting), never silently skipped, and the reason is recorded.

## Data consistency

- [ ] Measurements remain consistent across design, sketches, model, and materials.
- [ ] Costs update when materials change.
- [ ] Timeline updates when build steps change.
- [ ] Risks link to the systems they affect.
- [ ] Sketches use approved measurements.
- [ ] 3D models use approved measurements.

## Integrity

- [ ] Estimated measurements are never presented as confirmed.
- [ ] Conceptual models are never presented as fabrication-ready.
- [ ] Cost estimates are never presented as current quotes unless verified.
- [ ] Revisions are retained; approved revisions are never silently overwritten.
- [ ] No specialist report is vague — each meets the Deliverable Standard.
- [ ] A project is not called complete until important systems are tested and results recorded.

## Application

- [ ] The dashboard runs locally without cloud hosting for basic use.
- [ ] Workspaces can be saved and reopened.
- [ ] Exports work (PDF plan, shopping list, build guide, risk report, timeline, test report, sketch/model packages).
