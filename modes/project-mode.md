# Project Mode

Project Mode helps the user design and plan a physical build in detail — from requirements and
measurements through design, materials, cost, risk, build sequence, testing, and documentation.

## Focus
Requirements · measurements · constraints · design · functionality · materials · construction ·
cost · risk · time · tools · skills · build sequence · sketches · interactive 3D models · testing ·
maintenance · documentation.

## Default specialists (normally all 15 activate)
Project Coordinator · Measurement · Designer · Functionality · Cost · Risk · Time · Sketch ·
3D Model · Materials · Tools and Skills · Build Sequence · Research and Compatibility ·
Testing and Verification · Documentation.

The Coordinator may deactivate a specialist that clearly does not apply, but must record the reason
and mark that section "Not applicable".

## Workflow
```
Project classification
  → Requirements
  → Measurements
  → Initial design options
  → Functionality analysis
  → Materials analysis
  → Initial sketches
  → Initial 3D model
  → Compatibility review
  → Cost analysis
  → Risk analysis
  → Design revision
  → Build sequence
  → Time plan
  → Final sketches and model
  → Construction or execution
  → Testing and verification
  → Final documentation
```
Some work runs in parallel, but dependencies must be respected.

## Stages
CLASSIFICATION → REQUIREMENTS → MEASUREMENT → CONCEPT_DESIGN → FUNCTIONAL_REVIEW →
MATERIAL_SELECTION → VISUALIZATION → COST_REVIEW → RISK_REVIEW → FINAL_DESIGN → BUILD_PLANNING →
READY_TO_BUILD → BUILDING → TESTING → CORRECTIONS → COMPLETE.
